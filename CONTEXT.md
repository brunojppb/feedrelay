# FeedRelay

A single-user service that turns iPhone Shortcut taps into Instagram posts scheduled through Buffer, sourcing photos from a personal Immich library.

## Language

**Run**:
A single execution of the relay, triggered by one Shortcut tap. Captures the full lifecycle: smart-search → filter → caption → Buffer call. A run may or may not produce a [[Post]] (it can fail at any stage). Only one Run is ever in flight at a time — a second tap while a Run is `queued` or `running` returns 409 with the existing run id.
_Avoid_: Job, pipeline run, post job, pipeline execution

**Post**:
A row representing an Immich asset that has been successfully handed off to Buffer's `createPost` mutation (we hold a `buffer_post_id`). FeedRelay never observes the actual Instagram publication; "Post" here means "scheduled with Buffer". Used as the permanent dedup record: an asset that became a Post is never reused.
_Avoid_: Posted asset, scheduled post, placement, published post

**Asset**:
An item in the Immich library, identified by `immich_asset_id`. The unit of work the relay picks from. Becomes a [[Post]] only after a successful Buffer mutation; failed runs leave the asset eligible for re-selection (deliberate trade-off — see [[ADR-0001]]).
_Avoid_: Photo, image, media (use "media" only for the `pending_media` short-lived URL mapping)

**Candidate**:
An [[Asset]] returned by Immich smart search that is eligible to be selected for a [[Run]]. To qualify, the asset must (a) carry usable EXIF with at least one of city/country populated (no location → no candidate, since the post would be orphaned), (b) pass the layered face-area filter (per-face *and* combined thresholds), and (c) contain no recognized/named person regardless of face size. Named-person exclusion is stricter than unknown-face exclusion by design.
_Avoid_: Result, hit, match

**Caption**:
The text generated by OpenAI describing an asset, plus its hashtags and alt_text. Cached by `(asset_id, prompt)` so that retries within a [[Run]] (or future runs that re-pick the same asset after a Buffer failure) don't pay OpenAI twice. The cache is the only stateful checkpoint a run preserves across retries — everything else (smart search, filter, Buffer call) is re-executed from scratch.
_Avoid_: Description, blurb, post text
